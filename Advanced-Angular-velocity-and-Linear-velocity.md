在阅读本指南之前，请先阅读[高级前置条件](Advanced-Prerequisites)页面。

角速度用于根据速度准确旋转物体，如果实现得当，它将完美运行。实际上，如果你愿意的话，你也可以通过角速度旋转非球形物体，但你必须用一个可想象的球体作为基础。因为结果被认为是一个标量，所以它可以很容易地应用于任何速度。

## 什么是角速度？
在物理学中，物体的角速度是相对于时间其角位移的变化率。我想这些词对一些人来说太花哨了，所以让我们分解一下。你只需要理解`角速度`这个词。

想象一个球状物体，比如一个球，当这个物体移动时，它也在围绕其中心旋转或公转。角速度是球状物体的速度与围绕其中心旋转之间关系的测量。这些围绕中心的旋转通常被称为`转动`，一次转动是围绕中心的一次完整旋转。

我们可以查看角速度的国际单位制来进一步理解，即`弧度每秒`。我们知道一个完整的圆是2π，或360度。因为一次转动是完整的一圈，我们可以这样可视化：`x转动/秒 * 2π弧度/转动`。如果这看起来令人困惑，想想以下：我们每秒有x次转动，每次转动等于2π。这意味着每次转动我们有2π弧度。所以，这可以改写为：`x * 2π弧度/秒`，希望这对你来说是直观的。

在继续之前举个例子，想象每秒做5次转动。我们知道一次完整旋转是2π弧度，所以1次转动是2π弧度。这意味着每次转动我们旋转2π弧度，所以每秒5次转动我们每秒旋转10π弧度。这个关于_我们围绕中心点旋转得有多快_的测量值就是我们所说的_角速度_，有时也称为_角速度_。这里的角速度是`10π弧度/秒`，角速度是`10π`。

## 设置
在我们开始旋转之前，我们应该为角速度设置一个辅助属性。记住我们的角速度是每秒转动数，即每秒围绕中心旋转的总次数。为了将这个与我们的速度关联起来，我们需要找到一种利用速度向量的方法。我们可以通过取速度的大小（长度）来实现。这样做我们得到：`2π * 速度大小`。这定义了我们的角速度。让我们实现它：（注意这展示了c#7 getter）
```cs
double AngularVelocity
{
  get => Math.PI * 2 * velocity.Length;
}
```
因为Terraria有有用的辅助工具，你可以用`MathHelper.TwoPi`替换2π作为常量值，这比持续计算要快得多（因为TwoPi是一个常量）
```cs
double AngularVelocity
{
  get => MathHelper.TwoPi * velocity.Length;
}
```

## 应用角速度
你可以用很多不同的方式应用角速度，因为在Terraria中你可以用很多不同的方式应用旋转。一种方法是在AI中通过将角速度添加到实体的旋转上。这对NPC或弹射物很有用：`this.rotation += this.angularVelocity;`。重要的是你要将角速度_添加_到旋转上而不是直接设置它，因为记住它定义的是角_位移_。

如果你使用其他东西，你可能应该存储你自己的角度变量。然后在任何更新调用中，你可以应用与上面相同的行为。

对于自定义绘制代码，你应该将偏移角度传递给它进行绘制。以下示例假设你已经在某处应用了上述行为。这意味着你已经应用了角位移：`rotation += angularVelocity`。
```cs
spriteBatch.Draw
(
  texture, position, frame, color, rotation, size, scale, spriteEffects, layer
);
// rotation应该等于旧rotation加上角速度
```
要注意，如果你在任何地方都没有通过角位移更新实际rotation，而是传递`rotation + angularVelocity`给绘制调用，你将得到故障问题。

## 用数学推导线速度公式
角速度 = 角速度，用小写omega(w)表示 => `x转动/秒`

线速度公式：`v = s/t`，然后限定s：`s = 弧长 = 2π * r * (theta/360)`
其中r是半径，theta是角度。现在我们可以用弧度重写：`2π * r * (theta / 2π)`，2π互相抵消，留下：`r * theta`

所以现在我们有：`v = 线速度 = r * theta / t`。回到角速度：`w = theta / t`。这个公式你应该能理解，因为我们之前说过角速度是相对于时间的角位移。这正是这里所展示的。

所以我们得到：`v = r * w`
